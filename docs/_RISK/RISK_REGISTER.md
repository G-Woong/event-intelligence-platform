### R-ProviderDateWindowFidelity · 요청한 date window 가 provider 응답 결과를 제약하지 못함(out-of-window·주제 무관 기사 반환·메커니즘 미확정)  — Severity: MEDIUM (신규 2026-06-29 · ADR#84 — 첫 실 date-pinned bounded live run 이 드러낸 실 present blocker)
- Area: date-pinned live acquisition / provider date filter + query relevance / window enforcement
- Description: ADR#84 첫 실 live run(operator event=U.S. Supreme Court asylum metering ruling·occurrence_date 2026-06-25·window [2026-06-25, 2026-06-26]·guardian×nyt)에서 **URL 이 정확히 `from-date=2026-06-25&to-date=2026-06-26`(guardian)·`begin_date=20260625&end_date=20260626`(nyt)로 구성됨을 오프라인 캡처로 검증**했으나, 실 응답은 양 provider 모두 **2026-06-28(당일) 기사 10+10**을 반환했다(Austrian GP·World Cup·Iran 등 asylum 판결과 **주제도 무관**). 즉 cross-source 비교쌍 100개는 pinned window 가 아니라 *무관한 동일자 최신 뉴스*였고, near-match 0(max jaccard 0.0625)은 same-event recall 한계도 query 장문성(간결 hint 동일 결과로 배제)도 아닌, **요청 window 가 실제 결과를 제약하지 못한** 데서 비롯됐다(plumbing 정상). **단 메커니즘은 미확정**(adversarial MEDIUM-1): 반환 기사가 날짜뿐 아니라 **주제도 무관**하다는 점은 단순 date-filter 무시뿐 아니라 **`order-by=newest`+느슨한 `q` 매칭이 "최신-무관" 기사를 지배**(relevance/ranking)하는 가설을 동등 이상으로 시사한다 — 후보 가설=(i) provider 가 date-filter 무시, (ii) order-by=newest 지배, (iii) 느슨한 q OR-매칭, (iv) 이 토픽의 in-window 보도 0. **결정적 통제실험**(같은 쿼리에서 date 파라미터 유/무 결과 비교 + order-by 제거)은 **미수행**. 위험은 이 미확정을 *확정 "window 무시"*로 단정해 다음 단계를 오도하거나, out-of-window 기사를 same-event 비교쌍으로 오염시키는 것이다.
- 경계: R-DeterministicNearMatchRecall(detector recall 한계)·R-ProviderPairNarrowness(pair 다양성)와 달리, 본 RISK 는 *요청한 절대 date window(및 query relevance)가 provider 응답 결과를 실제로 제약하는지*만 본다.
- Current mitigation: `run_provider_query(enforce_window=True)` 신설 — provider 응답이 window 밖이어도 adapter 가 `published_at` 으로 [from_date, to_date] 밖 record 를 drop(전부 drop 시 `no_in_window_records`로 진짜 0 과 정직 구분). date-pinned executor 가 `enforce_window=True` 로 opt-in(기본 False=ADR#62~#82 동작 byte-보존). 잠금: test_provider_query_adapters(in-window 유지·out-of-window drop·no_in_window_records vs no_records). **단 enforce_window 는 date 증상만 보정한다** — relevance/order-by 차원(가설 ii/iii)은 미해결이라 **in-window 라도 주제 무관 기사면 same-event 후보는 여전히 0 일 수 있다**·또한 정상적 지연보도(D+2)도 좁은 창에서 잘릴 수 있다(precision↔recall). enforce 적용 후 in-window same-event 수율은 **미측정**(이번 run 은 enforce 전 실행).
- Closure: **통제실험으로 메커니즘(date-filter vs order-by=newest vs q-relevance vs coverage)을 분해**하고, enforce_window 적용 live run 이 *in-window cross-source coverage 가 실재하는 event*에서 same-event 후보 수율을 측정하며 provider date-filter/ranking 의미가 문서화될 때 부분종결·**메커니즘 미확정을 '무시' 확정으로 단정 금지·out-of-window 기사를 date-pinned 비교쌍으로 사용 금지·enforce_window 미적용으로 date-pin 측정 금지**.
- **ADR#85 부분진전(종결 금지)**: `provider_date_window_fidelity.py` 5-variant **통제실험 실행**(Guardian 5·NYT 4·paced·gate-respected·secret 0·raw body 0). 결과 전 비교 variant 동일(반환 전부 당일 2026-06-29·요청 window 밖·overlap 0.107/0.044): **date_param weak**(original==no_date)·**order_newest weak**(relevance==newest)·**query_relevance weak**(exact_phrase==original)·coverage zero_in_returned → **가설 (ii) order-by=newest 지배·(iii) 느슨한 q 를 약화(이 run 비차별 lever·newest-dominance 미배제)**·**`date_filter_ignored` 를 leading 가설(medium·cross-provider 일관)** 으로 좁힘(original 이 자기 요청 window 밖 반환 + original==no_date 가 'date filter 작동+coverage 0' 을 반증). **단 (i)date_filter_ignored 와 (iv)zero_in_window_coverage 는 완전 미분리**(단일 event·window-honoring control 부재)·**confidence medium(절대 확정 아님)**. 추가 분리는 window-honoring source(Federal Register·ADR#86) 또는 in-window 보도 실재 event 필요.
- **ADR#86 부분진전(종결 금지)**: window-honoring control 확보 — Federal Register adapter 실배선 + bounded live smoke 로 **FR date filter 가 응답을 실제 제약함을 실측**(operator window [2026-06-25, 2026-06-26]: "asylum metering" 31건→필터 적용 0건·"enforcement" **25/25 in-window·out-of-window 0·live_verified**). 이는 **Guardian/NYT 의 date_filter_ignored 가설을 보강**한다 — 같은 window 에서 window-honoring official source 는 in-window record 를 반환하므로 Guardian/NYT 0-in-window 가 "전역 zero coverage" 보다 "(news 측) date_filter_ignored" 쪽으로 기운다. **단 FR=regulatory 도메인 ≠ news 도메인**이라 news 측 (i)vs(iv) 완전 분리는 아닌 **보강 신호**(정직)·**confidence 여전히 medium**. news 측 최종 분리는 같은 window 의 in-window news 보도가 실재하는 event 필요.
