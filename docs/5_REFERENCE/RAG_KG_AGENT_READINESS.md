## 4b. 제품 출력 계약 — raw source ≠ public product (substrate→intelligence unit)
이 제품은 **뉴스 본문 하나를 그대로 보여주는 웹이 아니다.** 모든 소스(article/catalog/market/community/official/search)는 최종적으로 다음 중 하나로 정제된다:
- **Event substrate 의 근거**(publishable: official/article → Event CREATE/APPEND 근거) · **signal**(market/numeric → 비발행 지표) · **catalog/entity enrichment**(catalog_metadata → 비발행·KG/entity 보강 후보, ADR#40) · **held/review evidence**(약신호 corroborator → possible link) · **expansion seed**(search → URL 후보) · **public intelligence unit 의 구성요소**.
규칙: **raw article/catalog/market/community 를 그대로 public 으로 내보내지 않는다.** public 출력 단위는 사건 단위 Event(향후 curated Intelligence Unit). source-type publish gate(ADR#33/#35)+catalog fidelity(ADR#40)+cross-batch identity(ADR#40 병합·ADR#41 후보)+held 정책(ADR#38)이 이 계약을 코드로 강제한다. Agent/RAG/KG 는 이 substrate **위에서** 관련 source 확장수집·entity 연결·반응/영향/시간흐름 정리·citation 유지로 intelligence unit 을 생성한다(현재 미구축 — §2/§5). **계약 단일 출처: `INTELLIGENCE_UNIT_CONTRACT.md`**(IU 구성·source role·entity/semantic identity hook).

## 6b. LLM / Agent 진입 조건 (ADR#54/#55/#56/#57/#58/#59 — substrate gate, 본경로 구현 아직 0)
> 이 제품의 최종 주관 엔진은 LLM/Agent 다(여러 source 관측을 사건 단위로 묶어 연관도·공식성·커뮤니티 반응·시장 신호·카탈로그/엔티티 맥락·시계열 변화·불확실성을 정제한 **Intelligence Unit** 생성). 그러나 **raw source 를 즉시 요약해 public output 을 만드는 방식은 금지** — LLM 판단은 orchestration loop **내부**에서 evidence/guard 에 묶여야 한다. 아래 9조건이 **모두** 충족되기 전엔 LLM/Agent 본경로 진입 금지(현재 No-Go). 상태는 `real_source_smoke_report.agent_readiness_conditions` 가 결정론으로 산출(단일 출처).

**LLM/Agent 진입 전 최소 조건(9) — 현재 status:**

### 6b-R. ### 6b-R1d. R1 live query executed + cross-source near-match gap (원인 미확정) (ADR#77 — Lane A/B/C/D)

- **live run 결과(실측·1회):** `live_call_performed=True`·providers_ready=[guardian,nyt]·**`live_candidate_count=100`**(cross-source same-date **publishable 비교쌍**·≈10×10 Cartesian·≠same-event match)·**`publishable_pair_count=0`**(near-match 후보)·`production_candidate_status=blocked_no_publishable_pairs`·root cause **`no_title_overlap`**(near 0·fingerprint 0). **원인 미확정** — (i) 같은 사건의 cross-source paraphrase 를 결정적 recall 이 놓침, 또는 (ii) broad topic("central bank rate decision") 아래 서로 다른 사건/측면이라 정당하게 안 겹침(ADR#64 가 이미 "n=2 로도 구분 불가·100 쌍=비교대상" 명시). **detectable same-event overlap 은 여전히 0**(near/fingerprint)→overlap scarcity 는 해소가 아니라 **재확인**(R-SourceOverlapScarcity closure ≥1 미충족). production_gold_count 0·candidate_provenance=none(둔갑 0) 유지.
- **adjudicator-zone 연결(§4 ④·조건부):** **만약 (i)** 이면 이 갭은 §6b-R 매트릭스 **R3 embedding scorer / INTELLIGENCE_UNIT_CONTRACT §4 ④ semantic adjudicator**(embedding/LLM/KG)가 메우도록 설계된 영역이고, **(ii)** 이면 레버는 targeted same-event 좁은 쿼리다 — 어느 쪽인지가 미해결. 어느 경우든 runtime 은 No-Go(embedding/LLM 실호출 0·자동 same_event 0·MERGE_GATE/gold 전). live run 은 near-match 단계의 0 을 **실측**할 뿐 원인을 확정하지 않는다.
- **분류 정직성 fix(test-locked·adversarial 근거):** `blocked_no_publishable_pairs` 의 primary next_action(internal ops `next_manual_action`)을 사유로 분기 — `live_candidate_count>0` + `no_title_overlap` 이면 **원인 양가(paraphrase OR different-events·미확정)** 안내(좁은 topic/window 재실행으로 구분·(i)면 gated adjudicator·(ii)면 targeted), 그 외엔 role-guard 안내. live run 이 운영자에게 "source role guard 약화"로 오안내(publishable 비교쌍 100 존재)하던 결손을 교정하되 **단정적 paraphrase 로도 승격하지 않는다**. `_primary_next_action`·테스트 3종 잠금.
- **acquisition strategy(Lane C·다음 턴 bounded·무단 대량 호출 0):** ① **좁은 same-event-targeted topic/time-window 변주**(특정 breaking 사건·1d→3d/7d)로 (i)/(ii) 구분 — 일부 쌍이 hard band[0.2,0.5)에 드는지(같은 사건 후보의 약한 증거) ② provider record depth 확대 ③ (i) 확인 시에만 gated adjudicator(R3)가 lever ④ provider pair 확장은 legally/technically 가능한 publishable(official+news)만 ⑤ search=URL candidate only·community/market/catalog=anchor 0(source role guard 유지·약화 0). 본 run 은 host_gate=None 으로 실행됐으나 host 당 1회라 min-spacing 위반 구조적 불가 — 반복/sweep 시 host gate 주입은 다음 증분(R-CrossSourceNearMatchGap·R-ProviderPairNarrowness).

### 6b-S. Source-specific high-quality storage strategy (ADR#74 — Lane D · docs only · runtime No-Go)
> 최종 IU 는 raw source 직노출이 아니라 source 역할별로 정제·적재된 evidence/reaction/signal/entity 위에서 합성된다. 아래는 **storage 전략 방향**(runtime schema 구현 0·gate 통과 전 적재 금지) — source role guard(조건 2)를 storage 층까지 연장한다. **anchor 가 될 수 있는 것은 official/news 만**이며, 나머지는 역할이 고정된다.

| Source role | Role | Storage shape (방향) | KG edge eligibility | Risk |
|---|---|---|---|---|
| official/news | publishable evidence · **event anchor** | event evidence·canonical URL·published_at·title·source authority·update relation | event/entity edge (MERGE_GATE 후) | false merge·paywall/body 결손·source overlap 희소 |
| community | **reaction layer (anchor 아님)** | reaction summary·sentiment/vibe bucket·time-series volume·허용 시 대표 public snippet | reaction_to edge (event verified 후에만) | rumor amplification·ToS·identity leakage |
| market/numeric | **signal (anchor 아님)** | numeric observation·timestamp·instrument/entity link·anomaly flag | market_signal_for edge (entity/event verified 후) | spurious correlation·투자조언 오인 |
| catalog | **entity enrichment (anchor 아님)** | entity metadata·identifier·relationship·provenance | entity node/attribute (provenance 검증 후) | stale metadata·duplicate entity |
| search | **URL candidate (truth 아님)** | candidate URL·query·rank·discovery timestamp | none (fetch/validate 전까지) | search result volatility |
| unknown | **fail-closed** | quarantine only | none | 미분류 오염 |

반드시 명시: public IU 는 verified event/entity graph + source evidence + community reaction layer + uncertainty contract 이후에만 가능(R7). community 반응(느낌)은 중요하나 event anchor 가 아니라 reaction layer 다. **KG edge 는 provenance/confidence/source role 없으면 생성 금지**, GraphRAG retrieval 은 verified graph 전까지 금지. 본 §6b-S 는 *문서 전략*일 뿐 — storage runtime/schema 는 R4~R5 gate 통과 전 구현하지 않는다(R-SourceStoragePrematureSchema).
